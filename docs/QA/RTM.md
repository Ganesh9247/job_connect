# Requirements Traceability Matrix (RTM)

The Requirements Traceability Matrix maps the functional requirements of the JobConnect application to their corresponding Test Scenarios and Test Cases to ensure full test coverage.

| Req ID | Requirement Description | Test Scenario ID | Test Case ID | Status |
|---|---|---|---|---|
| **REQ_01** | Users shall be able to register an account with a unique email and specific role. | TS_01_01, TS_01_02 | TC_01_01, TC_01_02 | Not Run |
| **REQ_02** | Users shall be able to securely log in and receive a JWT token. | TS_01_03, TS_01_04 | (To be added) | Not Run |
| **REQ_03** | Job Seekers shall be able to view and search/filter job postings. | TS_02_01, TS_03_01 | (To be added) | Not Run |
| **REQ_04** | Job Seekers shall be able to apply to jobs and see their applications. | TS_02_03, TS_02_07 | TC_02_03 | Not Run |
| **REQ_05** | Job Seekers shall be able to upload a PDF resume. | TS_02_05, TS_02_06 | TC_02_06 | Not Run |
| **REQ_06** | Recruiters shall be able to post new jobs and view applicants. | TS_04_01, TS_04_03 | (To be added) | Not Run |
| **REQ_07** | Administrators shall be able to view and delete jobs and users. | TS_05_01, TS_05_02 | (To be added) | Not Run |
| **REQ_08** | The system shall enforce role-based access control (Authorization). | TS_01_06, TS_05_05 | TC_05_05 | Not Run |

*(This matrix should be updated continuously as new test cases are authored and executed.)*
